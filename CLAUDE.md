# CLAUDE.md

## Project

- TypeScript project
- TypeScript strict mode enabled

## Code Style

- Spaces for indentation
- Semicolons required
- OOP style preferred
- Consistent naming conventions throughout the codebase
- Add explanatory comments where logic isn't self-evident
- Keep file sizes manageable — split large files into logical modules
- DRY — extract shared logic into utilities rather than repeating code
- Code should be understandable, logical, and readable

## Workflow

- Test-driven development — write tests first, then implement
- Run tests before considering any task done
- Commit automatically when a task is complete
- Commit messages should be concise and informative
- Ask before making judgment calls (library choices, architecture decisions, restructuring)
- Do not rush to implement — confirm the approach first

## Communication

- Keep responses terse
- Be prepared to justify choices when asked
